# 🎯 NextGen LMS - Implementation Plan & Technical Prompt

קובץ זה מתעד את האסטרטגיה הטכנית ליישום המלצות השלד. השתמש בתוכן זה כפרומפט עבור ה-AI בכל שלב של פיתוח התשתית.

---

## 🚀 הפרומפט המרכזי ליישום (Master Implementation Prompt)

**המטרה:** שדרוג השלד הקיים לפלטפורמת SaaS מאובטחת, מהירה ודינמית.

### 1. ניהול נתונים (Data Layer)
- הטמעת **TanStack Query (React Query)** לניהול Caching וביצועים.
- הגדרת סכימות אימות בעזרת **Zod** להבטחת תקינות נתוני הסילבוס (Single Source of Truth).

### 2. אבטחה (Secure Infrastructure)
- פיתוח **Middleware** ב-Server-side לבדיקת הרשאות (RBAC) לפני טעינת הדף.
- בידוד הרצת קוד (CodeRunner) בתוך **Sandboxed Iframes** למניעת פרצות אבטחה.

### 3. ארכיטקטורה גנרית (SaaS Core)
- בניית **Universal Content Provider** התומך בטעינה היברידית (Local Files / Remote DB).
- יישום **Lazy Loading** ו-Suspense לחוויית משתמש חלקה בשיעורים עמוסי תוכן.

### 4. ממשק ניהול (Admin Control)
- בניית מערכת טפסים דינמית (Zero-Code) להוספת שיעורים וניהול תלמידים.
- הטמעת **Next-Themes** לניהול מצבי יום/לילה וצבעי מותג (Branding).

---

## 🛠️ הנחיות טכניות למפתח / AI:

1. **עבוד בצעדים קטנים:** אל תעבור לשלב הבא לפני שביצעת בדיקה (Test) לשלב הנוכחי.
2. **אבטחה מעל הכל:** לעולם אל תחשוף נתונים של תלמיד אחד לאחר. השתמש ב-Row Level Security.
3. **ביצועים:** השתמש ב-Server Components במידת האפשר כדי לצמצם את גודל ה-JavaScript הנשלח ללקוח.
4. **עצמאות:** וודא שכל קורס חדש שנוסף בתיקיית `content` מזוהה ונרשם אוטומטית במערכת.
